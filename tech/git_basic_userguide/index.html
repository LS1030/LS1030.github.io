Git基础使用不完全指北

 2022.2.23  2022.7.14  Tech  3444  7 分钟

#简介

Git是一个分布式版本控制系统，可以在本地与远程端均部署一个仓库，本地与远程之间可以同步更改。

Git的特点：

  1. Git直接记录快照，而非差异比较： 在Git中，每当你提交更新或保存项目状态时，它基本上就会对当时的全部文件创建一个快照并保存这个快照的索引。 为了效率，如果文件没有修改，Git 不再重新存储该文件，而是只保留一个链接指向之前存储的文件。Git对待数据更像是一个快照流。
  2. 近乎所有操作都是本地执行：在Git中的绝大多数操作都只需要访问本地文件和资源，一般不需要来自网络上其它计算机的信息。仅仅在与远程端同步的时候需要网络操作。
  3. Git保证完整性：Git 中所有的数据在存储前都计算校验和，然后以校验和来引用。
  4. Git一般只添加数据：你执行的Git操作，几乎只往Git数据库中添加数据。你很难使用Git从数据库中删除数据，也就是说Git几乎不会执行任何可能导致文件不可恢复的操作。

#基础使用流程

#Git的概念

Git有三种状态，你的文件可能处于其中之一： 已提交（committed）、已修改（modified） 和 已暂存（staged）。

  • 已修改表示修改了文件，但还没保存到数据库中。
  • 已暂存表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。
  • 已提交表示数据已经安全地保存在本地数据库中。

对应的，Git 项目拥有三个阶段：工作区、暂存区以及Git目录。

  • 工作区是对项目的某个版本独立提取出来的内容。 这些从Git仓库的压缩数据库中提取出来的文件，放在磁盘上供你使用或修改。
  • 暂存区是一个文件，保存了下次将要提交的文件列表信息，一般在Git仓库目录中。 按照Git的术语叫做“索引”，不过一般说法还是叫“暂存区”。
  • Git仓库目录是Git用来保存项目的元数据和对象数据库的地方。 这是Git中最重要的部分，从其它计算机克隆仓库时，复制的就是这里的数据。

#Git的使用流程

基本的Git工作流程如下：

  1. 在工作区中修改文件。
  2. 将你想要下次提交的更改选择性地暂存，这样只会将更改的部分添加到暂存区。
  3. 提交更新，找到暂存区的文件，将快照永久性存储到Git目录。

如果Git目录中保存着特定版本的文件，就属于已提交状态。 如果文件已修改并放入暂存区，就属于已暂存状态。 如果自上次检出后，作了修改但还没有放到暂存区域，就是已修改状态。

简而言之，就是修改文件-提交暂存-提交更新。

在日常使用中，还会使用到在本地新建仓库(init)、在本地克隆远程仓库(clone)、拉取远程仓库(pull)、把本地仓库推送到远程(push)、合并(merge)、变基(rebase)、新建分支(branch)、标记(tag)、软回退(soft)、硬回退(hard)。

#使用示例

下面演示如何在本地新建一个Git仓库(Hello_Git)，新建一个文件，把该文件添加到暂存区，提交，新建一个分支，做一个标记，然后推送到 Github 上。

#命令行

  1. 新建文件夹(Hello_Git), 并在文件夹打开终端。
  2. 终端输入git init，可以通过git config user.name和git config user.email查询并修改当前用户的名字和邮箱。可以输入git config --list查看所有Git配置信息。
  3. 或者在目标仓库储存位置打开终端，输入git clone [URL]，把远程仓库克隆到本地。
  4. 在仓库中添加一个文件(Git_Basic_UserGuide.md)。
  5. 终端输入git add .添加所有文件到暂存区，或者git add Git_Basic_UserGuide.md添加仅指定目录或文件。
  6. 在完成一次添加后，可以通过git status -s查看git仓库状态。如果此时修改了已经添加到暂存区的文件，需要再次添加。
  7. 在所有文件都修改完毕后，添加到暂存区，然后输入git commit -m [message]把当前暂存区文件提交到本地仓库中。可以通过git log查询历史提交信息。
  8. 新建一个分支git branch [branch name], 用git checkout [branch name]切换到对应分支, 并做出一些修改，然后进行一次提交。
  9. 切换到master, 使用git merge [branch name], 将[branch name]分支的内容合并到master分支，此时master分支会多一次提交记录。也可以使用git rebase master [branch name]实现结果相同的操作。但通常使用merge。
  10. 如果发现合并的提交有错误，或者在合并冲突的时候未保留有必要的文件，则可以进行回退操作(reset)。回退有软回退(soft)和硬回退(hard)和混合(mixed)回退。默认使用混合即git reset --mixed [HEAD^]，混会回退将放弃暂存区的文件及之前提交的文件，工作区的文件保持不变，[HEAD^]指上一个版本。软回退使用git reset --soft HEAD^，放弃提交记录，但会尽可能地保留之前提交的文件，若暂存区没有与之前相同的文件即文件在上次提交后未添加到暂存区，则将之前提交的文件放入暂存区,工作区文件保持不变。硬回退则放弃工作区、暂存区和Git目录中上次提交的所有文件，将文件回退到对应的版本。
  11. 在对应的提交上进行标记git tag [tags]，一般[tags]的内容是版本号如v1.0.0之类的。
  12. 添加远程仓库git remote add origin [remote address]。
  13. 把本地仓库推送到远程。git push origin master：将本地的master分支推送到远程。注意，默认情况下，不会传送标签到远程仓库，必须显式地推送标签git push origin [tags]

#VSCode

在VSCode有非常方便的图形化操作，在安装了各种非常好的插件后，除非比较复杂的操作，一般就不需要使用命令行了。

#插件安装

  • Git History
  • git ignore
  • Git Lens
  • Git Graph

#操作步骤

  1. 新建文件夹, 并用VSCode打开文件夹。
  2. 在VSCode打开终端，输入git init初始化GIT仓库。可同时使用git config来设置相关参数。
  3. 或者打开VSCode新窗口，进入左边栏的源代码管理，点击克隆储存库，在弹出的输入框输入URL，并选择储存路径，克隆远程库。
  4. 在项目中新建一个.gitignore文件，输入需要忽略的文件。
  5. 新建一个文件，此时左边栏的源代码管理图标会出现更改提示，单击打开。
  6. 将会会看到更改的文件，单击文件，会出现文件更改记录，本地文件与上次提交后的文件比较。确认无误后，可点击加号把文件添加到暂存区，也可点击返回图标放弃更改。
  7. 将所有文件添加到暂存区后，可在上面的输入框输入提交信息，并点击输入框上方的对号进行提交。
  8. 提交后，在对号图标旁边有Git History和Git Graph的图标，可图形化查看提交记录。在Git History页面中，可对每一个提交很方便地进行hard、soft、+tag、+branch操作。
  9. 在页面的左下角，即底栏的左边，有一个写着分支名的图标，可用它进行切换分支、新建分支操作。
  10. 在左边栏的源代码管理中，还可以看到VSCode自带的以列表形式展现的commit、File History、branches、remotes、status、tags、serch&compare，不过不太好看就是了。
  11. 在左边栏的源代码管理的···中，可使用它进行推送、拉取、克隆操作。
  12. 在底栏的分支名图标旁边，有一个同步按钮，可将本地仓库与远程仓库同步。将本地提交推送到远程同时拉取远程仓库的提交，有可能会产生冲突，需要合并冲突。
  13. 合并冲突时，VSCode会按照文件列出每一处冲突的部分，可选择保留一方的更改或者全部保留。

#CLion/PyCharm

跟VSCode一样，Clion或者PyCharm这些IDE也有比较好的插件和图形化管理工具。

#插件安装

  • GitToolBox

#操作步骤

  1. 打开IDE(CLion/PyCharm)，然后新建一个项目。
  2. 或者直接点击从VSC获取，版本控制选择Git，输入URL，将远程仓库克隆到本地。
  3. 在新建项目后，打开菜单栏的VSC，点击创建Git仓库，然后选择本地仓库储存路径，一般是当前项目文件夹。（克隆已存在的远程库忽略此步骤）
  4. 此时可以打开终端，输入git config设置相关信息。
  5. 此时菜单栏的VSC已变成Git，在Git菜单内，可以实现提交、推送、更新项目、拉取等操作。
  6. 在项目中新建一个.gitignore文件，输入需要忽略的文件。
  7. 在项目中新建一个文件，IDE会提示将文件添加到Git，选择添加。
  8. 在左侧菜单栏的提交菜单中，勾选变更的文件，然后在下方的输入框输入提交信息，点击提交。
  9. 在底栏的Git菜单中，在左侧可以看到分支信息，在中间可以看到对应分支的Git历史记录，在右侧可以看到对应某次提交的文件变更信息，可以在对应的提交和分支上打开右键菜单进行签出(切换)分支、新建分支、回退提交等操作。
