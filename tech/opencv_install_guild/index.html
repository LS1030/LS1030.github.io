#简介

OpenCV的全称是Open Source Computer Vision Library，是一个跨平台的计算机视觉库。OpenCV是由英特尔公司发起并参与开发，以BSD许可证授权发行，可以在商业和研究领域中免费使用。OpenCV可用于开发实时的图像处理、计算机视觉以及模式识别程序。[1]

OpenCV的官方网站为https://opencv.org,包含发布页面、教程、文档等资料。

OpenCV源码托管在Github，有基础包opencv，同时还有一个扩展包opencv_contrib 。此外，在gitee上也有对应的镜像gitee opencv和gitee opencv_contrib

#python版本安装

OpenCV的python版本安装比较简单:

1

pip install opencv-python

或者

1

pip install opencv-contrib-python

或者

1

conda install -c conda-forge opencv

#C++版本安装（ubuntu）

C++版本的OpenCV需要编译安装，因此比较麻烦，大致的步骤是：

  1. 安装编译环境
  2. 配置cmake参数
  3. cmake build
  4. make build
  5. make install

另外，编译安装也可以一起安装python版本的包。

#安装依赖

OpenCV的依赖项更多的是显示图像的GUI库，比如GTK、QT等。OpenCV默认启用GTK，不启用QT。ubuntu默认已经安装好GTK3，但仍然需要安装libgtk-3-dev才能正常编译带GUI版本的OpenCV。但是，如果是在wsl2内的ubuntu，则默认未安装GTK，自行安装非常麻烦，可以选择放弃依赖GTK部分的功能（比如imshow），在OpenCV的编译选项里关闭GTK。

OpenCV可以不依赖任何第三方库读取bmp、hdr、sun raster、ppm、pgm、pbm、pfm格式的图片，默认安装第三方扩展以支持png、jpg、tiff、webp、jpeg2000等格式。[2]如果OpenCV检测到部分扩展未安装，则会进行下载，也可能导致下载失败，可以先自行安装对应的第三方库再安装OpenCV。

OpenCV的视频处理的可以直接使用linux内的v4l2库，但是，如果要更强的视频处理能力，需要使用ffmpeg库，建议直接安装完整的ffmpeg。但是，一些兼容性问题，直接通过apt安装的版本与OpenCV不兼容，因此需要手动编译安装，具体方法见下一小节。

如果需要安装python版的话，需要先再python环境下安装numpy：

1

pip install numpy

OpenCV还可以用许多计算库，比如说OpenBlas、Eigen。OpenBlas的配置比较麻烦，详见下一小节。

OpenCV还可以使用了intel的libva库来加速视频处理。可以直接通过apt安装。

因此，需要必要安装的第三方依赖项有：

  1. libgtk-3-dev
  2. numpy

一行命令：

1

sudo apt install libgtk-3-dev python3-numpy

建议安装的第三方依赖项有：

  1. ccache
  2. libva-dev
  3. libhdf5-dev
  4. libeigen3-dev
  5. libgflags-dev
  6. libgoogle-glog-dev
  7. tesseract-ocr
  8. libtesseract-dev
  9. libopenblas-dev
  10. libopenblas-base
  11. liblapacke-dev
  12. ffmpeg（手动编译安装）

一行命令：

1

sudo apt install ccache libva-dev hdf5-dev libeigen3-dev libgflags-dev libgoogle-glog-dev tesseract-ocr libtesseract-dev libopenblas-dev libopenblas-base liblapacke-dev

#下载OpenCV源码

新建一个文件夹，比如说Software，然后下载OpenCV源码。

在OpenCV官网的Release页面下载最新版本（截止到2022-07-19是4.6.0）的zip压缩包，也可以去官方Github的Release里下载对应的zip压缩包或者tar.gz压缩包。下载后，~/Software下新建opencv文件夹，并把压缩包内容解压到里面。

如果是安装了git，则可以使用git clone的方式下载源码，需要使用-b [版本号]来切换最新版本。在新建的文件夹~/Software下，打开终端，输入以下命令，则会新建一个opencv文件夹，对应的源码下载到里面。

1

git clone -b https://github.com/opencv/opencv.git

如果github访问比较慢，则可以使用gitee的镜像库：

1

git clone -b https://gitee.com/mirrors/opencv.git

如果需要扩展包opencv_contrib，则可以在扩展包的github仓库的Tags页面下载对应的zip或者tar.gz压缩包，下载完后，解压到~/Software/opencv_contrib文件夹。同理，也可以使用git clone 的方式下载源码，在~/Software下打开终端，输入以下命令：

1

git clone -b https://github.com/opencv/opencv_contrib.git

或者

1

git clone -b https://gitee.com/mirrors/opencv_contrib.git

#利用cmake配置编译选项

进入opencv_build文件夹，运行cmake ../opencv，就可以生成默认的编译配置，但一般情况下都需要对默认的选项进行更改，可以使用cmake-gui(需要安装sudo apt install cmake-qt-gui)或者是在cmake和../opencv之间插入-D[编译选项]的方式。

具体的参数含义可以参考OpenCV官方文档[3] 、参考资料2的cmake编译参数部分[4]、CMake编译opencv各选项的含义[5]。

一般来说，根据架构或者功能的选择，常用的配置参数有：

Option Default Description
WITH_OPENCL ON Multiple OpenCL-accelerated algorithms are available via so-called "Transparent API (T-API)".
WITH_OPENGL OFF OpenGL integration can be used to draw HW-accelerated windows with following backends: GTK, WIN32 and Qt. And enables basic interoperability with OpenGL, see OpenGL interoperability and OpenGL support for details.
WITH_GTK ON GTK is a common toolkit in Linux and Unix-like OS-es. By default version 3 will be used if found, version 2 can be forced with the WITH_GTK_2_X option.
WITH_QT OFF Qt is a cross-platform GUI framework.
WITH_OPENMP OFF OpenMP API relies on compiler support.
WITH_TBB OFF Threading Building Blocks is a cross-platform library for parallel programming.
BUILD_TBB OFF OpenCV can download and build TBB library from GitHub, this functionality can be enabled with the BUILD_TBB option.
BUILD_opencv_python3 ON Build python3 bindings. Python with development files and numpy must be installed.
INSTALL_C_EXAMPLES OFF Install C++ sample sources from the samples/cpp directory.
INSTALL_PYTHON_EXAMPLES OFF Install Python sample sources from the samples/python directory.

另外，如果需要opencv_contrib，则需要通过-DOPENCV_EXTRA_MODULES_PATH=来指定opencv_contrib所在的路径(相对路径即可)。

如果需要指定安装路径，则通过-DCMAKE_INSTALL_PREFIX=指定，一般安装在/usr/或者/usr/local/或者/usr/local/opencv。

现在附上参考的cmake配置脚本，放置在opencv_build下，然后运行即可。

// ~/Software/opencv_build
cmake -DCMAKE_BUILD_TYPE=RELEASE \
    -DCMAKE_INSTALL_PREFIX=/usr/local/ \
    -DOPENCV_EXTRA_MODULES_PATH=../opencv_contrib/modules/ \
    -DWITH_OPENCL=ON \
    -DWITH_OPENGL=OFF \
    -DWITH_GTK=ON \
    -DWITH_QT=OFF \
    -DWITH_TBB=OFF \
    -DWITH_OPENMP=OFF \
    -DBUILD_EXAMPLES=OFF \
    -DINSTALL_C_EXAMPLES=OFF \
    -DINSTALL_PYTHON_EXAMPLES=OFF \
    -DBUILD_opencv_python3=ON \
    -DBUILD_SHARED_LIBS=ON \
    ../opencv

若cmake生成失败或需要更改配置重新生成，则可删除CmakeCache.txt后重新运行。

另外，比较推荐用cmake-gui来进行配置，通过sudo apt install cmake-qt-gui安装即可。

#cmake构建时常见问题

#无法下载某些文件

在cmake构建时，会出现Download failed的警告Warning，解决方法是到~/Software/cmake_build/CMakeDownloadLog.txt里面，查看下载失败的文件的文件名 和 下载链接，以及下载后存放的位置，一般是从cmake_download里下载对应文件并改名，保存到~/Software/opencv/.cache。常见的下载失败的文件有：

  1. ippicv
  2. ade
  3. data
  4. wechat_qrcode
  5. xfeatures2d
#OpenCV与ffmpeg编译问题

OpenCV可以调用ffmpeg来作为视频处理的后端，大大提高视频处理的能力，但是OpenCV使用cmake生成配置时，经常会出现检测不到ffmpeg的问题，解决办法[6]如下：

  1. 不使用apt安装ffmpeg，需采用源码编译安装的方式安装

  2. 安装ffmpeg时，需指定pkg-config的路径，关闭pic，使能shared，即

    1

    ./configure --prefix=/usr/local/ffmpeg --pkgconfigdir=/usr/lib/  x86_64-linux-gnu/pkgconfig/ --enable-pic --enbale-shared --disable-x86asm

  3. 因为设定了--enable-shared，因此需要手动指定ffmpeg动态库的路径，在/etc/ld.so.conf.d/新建ffmpeg.conf，然后输入ffmpeg动态库的路径：/usr/local/ffmpeg/lib/，然后sudo ldconfig更新一下，即：

    1
2
3
4
5
6
7

    sudu vim /etc/ld.conf.d/ffmpeg.conf

# 进入/etc/ld.conf.d/ffmpeg.conf，并输入：
/usr/local/ffmpeg/lib/
# 保存退出，返回终端

sudo ldconfig

  4. 不知道什么原因，OpenCV在编译时未能正确识别到libavcodec的版本号LIBAVCODEC_VERSION_INT（但在cmake生成的时候可以），因此需要在~/Software/opencv/modules/videoio/src/ffmpeg_codecs.hpp:80:5加一句#define LIBAVCODEC_VERSION_INT 5939100（把cmake检测到的版本号填进去）。这种方法虽然比较ugly但能用，如果有其他解决方法麻烦提个issue告知，谢谢。

#OpenCV与OpenBLAS、Lapacke的配置问题

OpenCV和OpenBLAS、Lapacke之间的兼容性存在问题，需要进行一些设置，主要是参考opencv的issue#12957[7]:

  1. apt安装libopenblas-dev、libopenblas-base、liblapacke-dev
  2. 添加lapacke.h软连接: sudo ln -s /usr/include/lapacke.h /usr/include/x86_64-linux-gnu/lapacke.h
  3. 添加libcblas.so软连接: sudo ln -s /usr/include/x86_64-linux-gnu/libblas.so /usr/include/x86_64-linux-gnu/libcblas.so
  4. 在opencv/cmake/OpenCVFindOpenBLAS.cmake内添加Open_BLAS_INCLUDE_SEARCH_PATHS到/usr/include/x86_64-linux-gnu
  5. 在opencv/cmake/OpenCVFindOpenBLAS.cmake内添加Open_BLAS_LIB_SEARCH_PATHS到/usr/lib/x86_64-linux-gnu
  6. 添加链接参数-lcblas:CMAKE_EXE_LINKER_FLAGS=-lcblas[8]

#不同版本共存问题

现在OpenCV4.x版本和3.x版本都比较常用（很多老代码都是用OpenCV3.x），因此需要解决两个版本共存的问题。解决方式很简单，改一下CMakeList.txt的搜索路径即可。

现在我们在/usr/local/下有两个版本的OpenCV，分别为/usr/local/OpenCV_4.6.0和/usr/local/OpenCV_3.4.16。默认情况下，CMakeList.txt里find_package(OpenCV REQUIRED)会找到最新版本的OpenCV。在需要使用3.4.16版本时，需要进行如下修改：

1
2
3
4
5

set(OpenCV_DIR "/usr/local/opencv-3.4.16/")
find_package(OpenCV REQUIRED
            PATHS ${OpenCV_DIR}
            NO_DEFAULT_PATH
            )

在需要切换版本时把3、4行注释即可。
