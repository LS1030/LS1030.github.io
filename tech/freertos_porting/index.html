FreeRTOS移植

 2022.12.21  2024.1.8  Tech  2999  6 分钟

#简介

FreeRTOS是最常用的免费商用开源的 RTOS，代码在FreeRTOS/FreeRTOS-LTS，里面包含了内核FreeRTOS/FreeRTOS-Kernel和应用于其他物联网和嵌入式产品所需的一整套库。

Stm32cubeMX 可以一键式直接生成配套的 CMSIS_OS-FreeRTOS 代码，非常方便，因此 FreeRTOS 成为我最常用的 RTOS 系统。但是从零开始移植 FreeRTOS 并没有试过，因此可以今天列一下步骤。其实官网上已经有很清晰的移植指南，可以直接查看。

下面以stm32F103为例开始移植，先使用 stm32cubemx 生成 Makefile 工程文件。

#文件解析

代码根目录里面有用的文件有event_groups.c, list.c, queue.c, tasks.c, timers.c, stream_buffer.c, include, portable。

其中，事件组event_groups.c, 任务调度list.c, 消息队列queue.c, 任务管理tasks.c, 软件定时器timers.c, 流缓冲区stream_buffer.c，是通用源文件，与架构无关，直接复制使用即可。

include文件夹下是 FreeRTOS 的头文件，同样与架构无关，直接复制使用即可。

portable文件夹是编译器相关文件，与架构相关，在这里面MemMang内存管理可以直接权复制使用，具体配置可以在 FreeRTOSConfig.h 配置。MemMang内存管理，里面包含了 FreeRTOS 的五种内存管理模式。

除此之外，还有各编译器相关的文件，我们可以看到常用的GCC、IAR、Keil、MSVC-MingW。我们看一下GCC，里面有各个架构所需要的文件，比如ARM_CM0、ARM_CM3、ARM_CM3_MPU等。我们知道stm32F1是ARM Cortex M3架构的，应该选ARM_CM3，里面有port.c和portmacro.h，这两个文件。

#移植

模仿一下 stm32cubemx 自动生成的目录结构，新建/Middlewares/Third_Party/FreeRTOS/Source。

然后把上述FreeRTOS内核源码的文件都复制到Source。

#内核裁剪

FreeRTOS使用名为FreeRTOSConfig.h的配置文件进行自定义。每个 FreeRTOS 应用程序的预处理器包含路径中都必须有一个FreeRTOSConfig.h头文件。

FreeRTOSConfig.h可以从官方Demo获取模板然后进行自定义的裁剪，详细信息可以看官方文档Kernel>Developer Docs>FreeRTOSConfig.h，我们选择CORTEX_STM32F103_Primer_GCC的FreeRTOSConfig.h为模板。

FreeRTOSConfig.h为每个工程自定义的头文件，用于配置内核，因此其实并不属于内核的一部分，因此我们放到Core/Inc目录下，然后根据需求进行裁剪。

  if (xTaskGetSchedulerState() != taskSCHEDULER_NOT_STARTED)

static StaticTask_t xIdleTaskTCBBuffer;
static StackType_t xIdleStack[configMINIMAL_STACK_SIZE];

  *ppxIdleTaskStackBuffer = &xIdleStack[0];
  *pulIdleTaskStackSize = configMINIMAL_STACK_SIZE;

}

  * @brief  FreeRTOS initialization
  * @param  None
  * @retval None
  */
void MX_FREERTOS_Init(void) {

  /* USER CODE BEGIN RTOS_MUTEX */
  /* add mutexes, ... */
  /* USER CODE END RTOS_MUTEX */

  /* USER CODE BEGIN RTOS_SEMAPHORES */
  /* add semaphores, ... */
  /* USER CODE END RTOS_SEMAPHORES */

  /* USER CODE BEGIN RTOS_TIMERS */
  /* start timers, add new ones, ... */
  /* USER CODE END RTOS_TIMERS */

  /* USER CODE BEGIN RTOS_QUEUES */
  /* add queues, ... */
  /* USER CODE END RTOS_QUEUES */

  /* Create the thread(s) */
  /* definition and creation of defaultTask */
  osThreadDef(defaultTask, StartDefaultTask, osPriorityNormal, 0, 128);
  defaultTaskHandle = osThreadCreate(osThread(defaultTask), NULL);

  /* definition and creation of myTask02 */

  /* USER CODE BEGIN RTOS_THREADS */

  /* USER CODE END RTOS_THREADS */

}

/* USER CODE BEGIN Header_StartDefaultTask */
/**
  * @brief  Function implementing the defaultTask thread.
  * @param  argument: Not used
  * @retval None
  */
/* USER CODE END Header_StartDefaultTask */
void StartDefaultTask(void const * argument)
{
  /* USER CODE BEGIN StartDefaultTask */
  /* Infinite loop */
  for(;;)
  {
    osDelay(1);
  }
  /* USER CODE END StartDefaultTask */
}

  /* Infinite loop */
  for(;;)
  {
    osDelay(1);
  }
  /* USER CODE END StartTask02 */
}

#总结

FreeRTOS移植其实很简单，就是把源码添加到工程中，然后使用定时器终端给内核配置时钟，新建freeRTOS.c编写任务，然后在main函数内开启内核就可以了。
