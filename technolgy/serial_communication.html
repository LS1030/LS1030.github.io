串行通信

发表于 2019-10-21 更新于 2020-04-07 分类于 技术 阅读次数： 评论数：
本文字数： 3.1k 阅读时长 ≈ 3 分钟

简介

在远程通信和计算机科学中，串行通信（英语：Serial communication）是指在计算机总线或其他数据信道上，每次传输一个比特数据，并连续进行以上单次过程的通信方式。
串行通信的例子有：UART,I2C,SPI,RS-232,以太网，SATA。PCI-E等。

通信概念

串行传输与并行传输

传输可以分为串行与并行传输。

串行通信技术，是指通信双方按位进行，遵守时序的一种通信方式。串行通信中，将数据按位依次传输，每位数据占据固定的时间长度。

并行传输指的是数据以成组的方式，在多条并行信道上同时进行传输，是在传输中有多个数据位同时在设备之间进行的传输。

简单来说，串行传输就是只有一条路，数据按时并且有序传输，单次传输1个字节。如下图：

并行传输有多条路，数据同时传输，单次传输多个字节。如下图：

并行传输的优势：一次能同时传输多个数据。
并行传输的劣势：最高传输频率不高；容易受到干扰；需要多条数据通道。
并行传输的用途：仅用于短距离传输。

串行传输的优势：传输频率高，可达10Gbs；只需少量数据通道；抗干扰能力强。
串行传输的劣势：一次只能传输一个字节。
串行传输的用途：多用于长距离传输。

目前来说，串行传输有取代并行传输的趋势。

同步传输和异步传输

同步传输是指每一个数据位都是用相同的时间间隔发送，而接收时也必须以发送时的相同时间间隔接收每一位信息。不论是否传输数据，接收单元与发送单元都必须在每一个二进制位上保持同步，I2C和SPI是同步传输。

同步传输要求双方时钟同步，双方在约定的时间内对数据进行采样，一般以上升沿或下降沿作为开始或停止的信号，高电平或低电平时数据有效。

异步传输是指将比特串划分为一个个小组发送，这些小组可以在任何时间发送，UART和CAN是异步传输。

异步通信不需要时钟信号，双方依靠数据中同步用的信号位进行信号同步，或者以数据帧的形式进行传输，一般会有起始位及停止位，也会有数据校验位。

单工，半双工和全双工

单工：只能进行单方向的通讯。

半双工：双方可以互相通讯，但不可以同时接收与发送。

全双工：双方可以互相通讯，可以同时接收与发送。

波特率和比特率

比特率，指单位时间内传输送或处理的比特的数量，单位为“比特每秒”（bit/s或bps），如百兆网络的速度最大为100Mbps。

波特率，单位时间内载波调制状态变化的次数，也可以被理解为单位时间内传输符号的个数。

“比特率”有时可以与“波特率”交换使用，只有在一个数据传输系统的每一个调制转变刚好承载一个比特的时候正确。

物理层

电平标准

CMOS标准（0~Vcc，TX-RX，2线）：输出：逻辑0：0V，逻辑1：Vcc；输入：逻辑0:0~0.7Vcc;逻辑1：0~0.2Vcc。

TTL标准（0~5V，Tx-Rx，2线）：逻辑0：0~0.5V；逻辑1：2.4V~5V。

RS-232标准（+15~-15V，Tx-Rx，2线）：逻辑0：3~15V；逻辑1：-15~-3V。

RS-485（-6~+6V，差分，A-B，2线）：逻辑0：-6~-2V；逻辑1：2~6V。

RS-422（+15~-15V，差分，Tx+-~Rx+-，4线）：逻辑0：5~15V，逻辑1：-15~-5V。

特性

CMOS与TTL的特性：

  • TTL是电流控制器件，CMOS是电压控制器件。
  • 同样的电压下，COMS可以直接驱动TTL，但反之不可以，需要增加上拉电阻。
  • TTL延迟低，功耗高；CMOS延迟高，功耗低。

RS-232/485/422的特性：

  • RS485半双工，RS232和422全双工。
  • RS232抗干扰能力弱，RS485/422抗干扰能力强
  • RS232传输速率低（20Kbps），RS485/422最高传输速率10Mbps。
  • RS232传输距离短（最大50米），RS485/422最长传输距离1200米（9600bps）。
  • RS232仅允许一对一通信；RS485允许一对多通信，连接32/128/256个收发器，但是只能有一个主机；RS422主从通信，从设备不能互相通信，最多10个节点。
  • RS485/422需要终端电阻，RS485需要120Ω，RS422需要100Ω。

协议层

UART

UART通信一般使用TTL电平标准，点对点通信，通过Tx向Rx进行数据的发送，全双工，异步传输，需要提前约定好波特率，一般是9600或115200bps。

UART的一个数据包由起始位，数据位，校验位，停止位组成，如下图：

  • 起始位一般为逻辑0，1位。
  • 数据位可以有5、6、7、8位，一般8位。
  • 校验位可以有无校验（noparity）、奇校验（odd）、偶校验（even）、0校验（space）、1校验（mark）。
  • 停止位一般为逻辑1，可以有0.5、1、1.5、2位。

I2C(IIC)

I2C一般采用3.3V或5V电平，主从通信，同步传输，半双工，有以下特性：

  • 仅需要2条线，一条时钟线（SCL），一条数据线（SDA）
  • 支持多主机多从机通信
  • 每个设备都有自己的地址，地址是7位或10位。
  • 采取主机产生时钟并广播，从机应答的方式初始化通信，多主机同时广播时有仲裁机制，通过检测SDA的电平进行仲裁，低电平优先。
  • 通信速率有标准模式（100kbps）、快速模式（400kbps）、高速模式（3.4Mbps）。
  • 使用I2C时应开漏输出并且加上拉电阻

I2C的通讯流程如下：

通讯流程为：

  1. 主机发送起始信号
  2. 主机广播从机地址
  3. 主机发送读写位，若读写位为0，则为写；若为1，则为读
  4. 从机应答。
  5. 主机写时，主机发送数据，以字节为单位，每一个字节从机需发送应答位。主机读时，从机发送数据，每一个字节主机发送应答位。
  6. 传输完毕，主机发送停止信号。
  7. 若需要给不同从机传输数据，主机可发送重复起始信号持续传输，最后发送停止位。

总线空闲时，SCL和SDA均为高电平。

在总线空闲时，拉低SDA（数据线）产生起始信号，拉高SDA产生停止信号。

发出起始信号后，主机应拉低SCL并产生时钟。

在SCL（时钟线）为高时，数据有效。

应答位上0表示应答，1表示非应答。

SPI

SPI一般采用5V或3.3V电平标准，主从通信，同步传输，全双工，有以下特性：

  • 采用4根线，SCLK（时钟线），CS（片选线），MISO（主收从发），MOSI（主发从收）。
  • 一对多通信，从机无法互相通信。
  • 采用片选线选择从机
  • 通信速率理论上不作限制，实际上几Mbps左右

SPI的通讯时序如下：

通讯流程为：

  1. 主机拉低CS，通讯开始。
  2. 主机产生时钟。
  3. 主机从机开始传输数据，数据单位可选4位到16位，一个时钟周期传输1位。
  4. 主机拉高CS，通讯结束。

总线空闲时，CS为高电平。

总线空闲时，主机把CS拉低选中从机，同时产生时钟，准备开始传输。

SPI有四种传输模式，根据CPOL（时钟极性）和SPHA（时钟相位）来确定4种模式，规定了采样和触发时的时钟沿：

  • CPOL = 0，SCK在空闲时为低电平；SPOL = 1，SCK在空闲时为高电平
  • CPHA = 0，数据在奇数边沿采样；SPHA = 1，数据在偶数边沿采样，从1开始计数。
  • Mode0，CPOL = 0，CPHA = 0。
  • Mode1，CPOL = 1, CPHA = 0。
  • Mode2，CPOL = 0, CPHA = 1。
  • Mode3，CPOL = 1, CPHA = 1。
  • 一般采用Mode0和Mode3。

采样时，数据有效，应保持不变。触发时，数据发送，此时MOSI和MISO可以改变。

数据传输一般采用高位先行（MSB）模式。
